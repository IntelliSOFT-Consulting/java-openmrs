Linked List

This tutorial develops a custom collection class based on the linked list data structure (slides: PPTX PDF).

1. Creating the nodes

Linked lists work by using a holders (called nodes) to hold a data element and link to another holder. So we start by creating a node class which will initially be used to hold strings. The class needs the following properties (remember to create getter and setter methods for the properties):

  • data (String)
  • next (same class as itself)

Create also a simple application to test this node class. It should create some instances of the node class, assign data (i.e. a string) to each one, and then link them using their next properties. Assuming the first node in the chain of linked nodes is called first, we can iterate through the chain using:

LinkedListNode1 current = first;
while (current != null) {
  System.out.println(current.getData());
  current = current.getNext();
}
Solution: LinkedListNode1.java, LinkedListTest1.java

2. Working with Object data items

In order to allow our linked list to store objects of any type - we should change the type of the data variables in the node class to Object instead of String. Then the test application can be modifed to add items of any type (e.g. int, boolean, String).

Solution: LinkedListNode2.java, LinkedListTest2.java

3. Creating the list class

Next we create the actual list class that will hold the nodes, and manage tasks such as adding items to the list. To hold all of the nodes the list class only needs a reference to the first node in the chain of nodes. However, in order to add items to the end of the list, it is convenient to also keep a reference to the last node in the chain. The list class should have an add(Object obj) method which works as follows (assuming the list class has node variables called first and last):

  1. Creates a new node object and sets obj as its data item
  2. If first is null, the list must be empty, so set first to reference the new node
  3. If last is not null, then set it's next reference to the new node
  4. Set last to reference the new node
Solution: LinkedList3.java, LinkedListTest3.java

4. Going generic

Next we modify our list and node classes to be generic, i.e. to allow the user to specify a type parameter rather than just use object references.

  1. Add a type parameter called T to LinkedListNode.
  2. Change the data member to be of type T and update the getData and setData methods.
  3. Add a type parameter called T to LinkedList and update the first and last node objects to use that type parameter.
  4. Update the add method of LinkedList to take a parameter of type T.
  5. Update the test application to use an instance of LinkedList<String>.
Solution: LinkedListNode4.java, LinkedList4.java, LinkedListTest4.java

5. Extra functionality

Finally we want to implement some extra methods on the LinkedList class to make it more useful. Try adding the following methods:

  1. size() - returns the number of elements in the list (hint: add a counter to count the calls to add).
  2. get(int index) - returns the data item at the specified index.

Test the size and get methods by looping through the list in your test application as follows:

for (int i = 0; i < list.size(); i++) {
  System.out.println(list.get(i));
}
Solution: LinkedList5.java, LinkedListTest5.java
