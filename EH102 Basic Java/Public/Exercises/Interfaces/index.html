Interfaces

This set of exercises explores Java interfaces - pure abstract classes.

1. Implementing a calculator

Implement the following interface to create a working calculator, and create a program which tests each of the 4 mathematical operations.

interface SimpleCalculator {
  public double add(double x, double y);
  public double sub(double x, double y);
  public double mul(double x, double y);
  public double div(double x, double y);
}
Solution: Exercise01.java

2. Multiple inheritance using interfaces

Although it is not valid in Java for a class to extend multiple classes, it is valid for a class to implement several interfaces.

  1. Create an interface called A which has
    • A data member called PI of type double
    • A method called foo
  2. Create another interface called B which extends A and adds
    • A method called bar
  3. Create a third interface called C which has
    • A method called hello
  4. Create a class which implements B and C. Observe that this class must define all the methods in A, B and C and has access to PI
Solution: Exercise02.java

3. Implementing a generic interface

The Arrays.sort(Object[] array) method can sort an array of any object type, provided that the object type implements Comparable. Comparable is a generic interface, so

  1. Create a class called Person with surname and forename properties of type String
  2. Make person implement Comparable<Person>
  3. Edit the compareTo method to make it compare two persons by comparing their surnames
  4. Create a Java program with an array of Person objects with different names
  5. Pass the array to Array.sort(...) before printing each item in the array to see the new order
Solution: Exercise03.java
