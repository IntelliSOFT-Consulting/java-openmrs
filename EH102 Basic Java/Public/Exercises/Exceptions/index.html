Exceptions

These exercises are designed to reinforce the concepts in the Exceptions lecture

1. Basic Exceptions

This is a simple program which demonstrates throwing and catching exceptions.

  • Create your own class called ReverseString and a main() which will be empty for now.
  • Create a static method called reverse. It takes an argument of a String and returns a String.
  • In reverse, check if the String has a length of 0. If the length is 0, the reverse method will throw an exception.
  • Now include the code to reverse the String. Do not use the String method reverse().
  • Now, in the main() method, attempt to call this method inside of a try block and deal with any potential exceptions in the catch. Include a finally statement that displays when main() has finished.
Solution: ReverseString.java

2. Errors

This exercise demonstrates catching an Error.

  • Create your own class called StackOverflow and a main() which calls a static method named myMethod.
  • Create a static method called myMethod. It takes no arguments and simply calls itself.
  • Wrap this call to itself in a try and in the catch, print out something to the console to show that you have a chance to do something before the Error shuts everything down. The toString method on the error will print out what type of error is occuring.
Solution: StackOverflow.java

3. Runtime Exceptions

Remember, runtime exceptions are unchecked. They are usually not handled, but instead, the program is modified to prevent them from occurring in the first place.

Create your own class which when run causes an ArrayOutOfBoundsException and run it.

Solution: ArrayOutOfBounds.java

Handle the ArrayOutOfBounds Runtime Exception by wrapping it in a try catch block and re-run your program.

Solution: ArrayOutOfBounds2.java

4. Custom Exceptions

In this exercise, you will create your own type of exception.

  • Create your own class called GoodHobbies and a main() which will be empty for now.
  • Create a class called BadHobbyException (can be in same file or same package) which extends Exception.
  • Create a method inside GoodHobbies called checkHobby which is NOT static takes a String arguments and throws a BadHobbyException if the hobby passed in is bad. Otherwise, print out that the hobby is good to the console. Create a list of good hobbies and add whatever you like.
  • In the main() method get the hobby out of the command line argument and create a GoodHobbies Object and call the checkHobby method with the hobby.
Solution: GoodHobbies.java

5. Exception Heirarcy and Rethrowing Exceptions

This exercise will combine many topics including java io, throwing and catching exceptions, the exception heirarchy, and rethrowing exceptions.

  • Create your own class called ReadFile and a main() which will call the method ReadFile passing one String argument, the name of the file to read which is myFile.txt, from within a try. This file does not exist on the filesystem.
  • Create a static method readFile which takes a String as the parameter.
  • Create a File object. From within a try block attempt to read the file using the classes FileReader and BufferedReader. See the Java API for details on how to use these.
  • Inside the catch, catch an IOException and a FileNotFoundException.
  • If the file is not found, attempt to create the file from within that catch. See the API for the appropriate method. If the attempt fails, rethrow the FileNotFoundException.
  • Run this program twice. The first time, there should be an exception because the file doesn't exist but the second time, the file will already be there because you created it when there was a FileNotFoundException the first time you ran the program.
Solution: ReadFile.java
