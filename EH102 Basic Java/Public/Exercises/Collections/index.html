Hashcodes and Collections

These exercises are designed to reinforce the concepts in the Collections lecture.

1. Overriding hashcode() and equals()

This is an exercise which demonstrates setting the hashcode and equals methods in a custom Object.

  • Make a Mountain class that has private instance variables name and height and public get and set methods to access height and name.
  • Override the Object's equals method in the Mountain class. If a Mountain has a name that is different from another Mountain's name then the equals method returns false. If they are the same, return true.
  • Override the Object's hashcode method by taking the mountain name and applying .hashcode().
  • Add 2 mountains with the same name and 1 mountain with a different name into the HashSet and print the contents.
  • Override the toString() method, using the name of the mountain.
Solution: Mountain.java

2. Comparable Interface

This exercise demonstrates using the Comparable Interface to sort Collections.

  • Modify your Mountain class so that the main method creates a TreeSet of mountains.
  • The Mountain class should implement Comparable. Compare the names of the mountain.
  • Add 5 mountains to a TreeSet and then output the TreeSet
Solution: Mountain2.java

3. HashMaps

This exercise demonstrates using HashMaps to store and retrieve key/value pairs.

  • Create a HashMap<Mountain,Integer>
  • Create a Mountain object which has a instance variable of name and set the name to "Killi".
  • Insert the element with the .put() method
  • Retrieve the Integer and output the the value to the console using the .get() method with the Mountain as your key.
  • Create a new Mountain() with the SAME name as your first mountain.
  • Retrieve the value from the map with the new mountain
Solution: Mountain3.java

4. Comparator Interface

In this exercise, you will use the Comparator Interface to sort Collections.

  • Create a class Patient with private instance variables name and address of type String and birthdate of type Date and weight of type int.
  • Create a constructor with these four variables as arguments
  • Create corresponding public getter and setter methods.
  • Implement toString() with name and birthdate.
  • Create a class PatientSort which implements a Comparator of Patients. Compare using the name Strings.
  • In the main method create an ArrayList of patients and add four patients.
  • Print them to the console unsorted, sort them, and then print out the sorted list.
Solution: Patient.java
