Generics

This set of exercises introduces generics.

1. Using a strongly typed ArrayList

It's valid in Java to use an ArrayList without specifying a type, i.e. ArrayList a = new ArrayList(); - although this will produce a compiler warning, because using any of the collection classes without using generics is not recommended.

  1. Create a Java program with a main method
  2. Create an instance of ArrayList<String> and an instance of ArrayList<int>
  3. Add some items to both arrays, observing that you can't add a string to an ArrayList<int> and vice versa
  4. Print some of the values to the console using get(...) to retrieve them from each array
Solution: Exercise01.java

2. Equality testing using generics

Supposing we develop a simple class to check the equality of two objects - we need the class to be flexible enough to work with any class, but we also want to ensure at compile time that the two objects being tested are of the same type. Without generics the class is defined as:

class EqualityTester1 {
  public boolean test(Object o1, Object o2) {
    return o1.equals(o2);
  }
}

It can be called with objects of different types, e.g. EqualityTester1.test("Hello", true)

  1. Create a new version of this class which uses a generic method
  2. Create another new version which is a generic class
  3. Create a Java program which tests the three different versions of this class
Solution: Exercise02.java

3. Multiple type variables

Generic classes are not limited to just one type variable.

  1. Create a class called Pair that takes two type variables (e.g. Pair<K, T>)
  2. Give it a member of each type, e.g.
    protected K first;
protected T second;
  3. Create a constructor which initializes both members from parameters
  4. Create getter methods for both parameters
  5. Create a Java program that tests the Pair class by creating instances with different types
Solution: Exercise03.java

4. Working with Maps

A map (sometimes called an associative array) is a special kind of collection which has a set of unique keys and a collection of values, and each key is associated with one value. The Map interface in Java takes two type variables - the first for the keys, and the second for the values.

  1. Create an instance of HashMap<String, Object>
  2. Add some values to the map of different types
  3. Print all of the values in the map using for (Object o : map.values()) { ... }
  4. Print all of the keys in the map using for (String s : map.keySet()) { ... }
Solution: Exercise04.java
