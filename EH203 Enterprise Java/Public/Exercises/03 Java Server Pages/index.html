EH203 - Exercises

Java Server Pages

This set of exercises introduces Java Server Pages

1. Hello World

This JSP simply prints something to the HTML response using out.println(...).

Solution: exercise01.jsp

2. Counter with helper class

This JSP uses a helper class to keep track of the number of page requests. The helper class is imported using a page directive such as

<%@ page import="eh203.jsps.Counter" %>

Solution: exercise02.jsp, Counter.java

3. Header and footer templates

It's very common to split pages into header, content and footer, so that the header and footer can be shared between many pages in a site, as a way of reducing duplicated code. One JSP's content can be included in another JSP using an include directive such as:

<%@ include file="exercise03header.jsp" %>

Solution: exercise03.jsp, exercise03header.jsp, exercise03footer.jsp

4. Counter with scriplet

Instead of using a helper class, implement a counter using a JSP declaration tag (<%! ... %>).

Solution: exercise04.jsp

5. Request parameters

Use a scriplet to generate an HTML table dynamically from the request parameters. You can enumerate the parameters as follows:

for (Object name : request.getParameterMap().keySet()) {
  String value = request.getParameter(name.toString());
  ...
}
Solution: exercise05.jsp

6. Initialization parameters

In order to define initialization parameters as we would with a regular servlet, we have to define our JSP "servlet" in web.xml:

<servlet>
  <servlet-name>Exercise06</servlet-name>
  <jsp-file>/exercise06.jsp</jsp-file>
  <init-param>
    <param-name>email</param-name>
    <param-value>rseymour@pih.org</param-value>
  </init-param>
</servlet>
Solution: exercise06.jsp
