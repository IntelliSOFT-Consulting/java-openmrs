Sessions

This exercise allows you to practice using Sessions. You will use an HttpSession object, cookies, url encoding, and forms to maintain session data in the http stateless environment.

Practice

Create a Dynamic Web Project in Eclipse.

Form-based Sessions

  1. Create a jsp file.
  2. Create a form with first name and last name fields. Make the form type POST.
  3. Create a servlet file.
  4. Have the servlet get the first name and last name out of the request object and greet the user by printing out html with the jsp variables in the place where name should be.
  5. Test that this works.

Cookie-based Sessions

  1. Set first name and last name cookies in the servlet.
  2. Modify your jsp file to display a greeting if you have cookies and to show the form if not.
  3. Test that this works.

Url Encoded Link-based Sessions and Cookie Deletion

  1. add a link to delete cookies on your jsp, sending the user to a second servlet (which you will create in the next step). This link should have a parameter, called remove and it should have a value of yes. Don't forget to URLEncode the link.
  2. Create a second servlet that just gets the first name and last name cookies if they exist and deletes them if the request parameter "remove" is "yes". Remember that this will be a request of type GET. Print out a message saying that the cookies have been deleted.
  3. Test that this works.

HttpSessions

  1. On your second servlet page (where you delete the cookies), comment out the code that prints out the message that your cookies have been deleted.
  2. Create an HttpSession object and store first name and last name in it.
  3. Create a third servlet page. Create an HttpSession object and get first name and last name from it.
  4. Go back to your second servlet and, after the HttpSession code you wrote, redirect from your second servlet to the third servlet you just created. Check google to learn how to response redirect from one servlet to another if you are unsure.
  5. In the third servlet, print out a message that shows which cookies are deleted, using the information you got out of the session on this servlet in a previous step.

SOLUTION: The project "Session_Project" will be put in this folder for you to check your answer once you have completed the assignment.
