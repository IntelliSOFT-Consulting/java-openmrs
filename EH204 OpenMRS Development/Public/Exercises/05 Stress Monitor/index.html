Stress Monitor: A new OpenMRS module

This tutorial creates a basic module from scratch, which will be used to let clinicians track the stress levels of the patients, and will warn them when a patient is stressed.

1. Creating a new empty module

When creating a new module, we never really start from scratch - its much quicker to take an existing module and rename it

  1. Get a copy of the basicmodule (svn)
  2. Rename all packages and classes to use appropriate names for a module called stressmonitor
  3. Update references in config.xml and other metadata files to use new module name and packages

Build the module using the ant script and deploy it in OpenMRS to check for problems.

2. Add a utilities to measure patient stress

In this project we will define stress as pulse × blood pressure, where blood pressure is systolic blood pressure (mmHg). This isn't medically realistic!

  1. Add a utility class with the following helper function: 
    double getPatientStressLevel(Patient patient)
    which will calculate the stress level of the specified patient
  2. Add a global property to the module called stressThreshold which is the stress level value above which a patient is considered "stressed".

3. Add extension to patient dashboard

Add a extension to the patient dashboard to show a warning message when a patient is "stressed". The extension should display nothing when a pateint's stress levels are normal. Build the module using the ant script and deploy it in OpenMRS to check for problems.

4. Add new stress graph tab

Add another extension which adds a new tab to the patient dashboard to display a graph of the patient's stress level. Use flot to create the graph.
